# 챗노트 (ChatNote) - 서비스 개요

> 📌 채널 URL: http://pf.kakao.com/_IHxegn

## 1. 핵심 문제

### 현재 사용자 행동
```
유튜브 보다가 "나중에 봐야지" → 카톡에 링크 던짐
맛집 발견 → 카톡에 링크 던짐
갑자기 아이디어 → 카톡에 메모
```

### 문제점
- 쌓이기만 하고 **정리 안 됨**
- 나중에 **"그거 뭐였지?"** 못 찾음
- 정리하려면 **귀찮음** (복붙, 수동 분류)

### 인문학적 통찰
> 사람은 **존나 귀찮은 동물**이다.
> 저장할 땐 "일단 던지고 나중에 보지 뭐" (1초)
> 정리는? "언젠간 하겠지..." (안 함)

---

## 2. 해결책

### 핵심 아이디어
**"나와의 채팅방" 대신 → "메모봇" 카카오 채널 사용**

```
나와의 채팅방: 보내기만 됨 ❌ 읽기 불가
카카오 채널:   보내기 ✅ + 스킬서버로 수신 ✅
```

### 사용자 경험
| 행동 | 기존 | 챗노트 |
|------|------|-----------|
| 저장 | 나한테 던짐 (1초) | 메모봇한테 던짐 (1초) |
| 정리 | 수동... (안 함) | **자동!** |
| 검색 | 스크롤... (포기) | AI한테 물어봄 |

**습관 거의 안 바뀜 + 완전 자동화!**

---

## 3. 핵심 가치

### 1. 습관 그대로
- 똑같이 1초 만에 던지기
- 채팅 상대만 바뀜 (나 → 메모봇)

### 2. 자동 정리
- AI가 즉시 분류 (영상, 맛집, 쇼핑, 할일 등)
- 태그 자동 생성
- 요약 자동 생성

### 3. 쉬운 검색
- "저번에 저장한 맛집 뭐였지?" → 바로 답변
- 카테고리별 조회
- 기간별 요약 리포트

---

## 4. 서비스 구성

### 4.1 카카오 채널 챗봇 (메모 수집)
- 사용자가 메모 던지는 곳
- 스킬서버로 메시지 자동 수신
- 즉시 정리 피드백 제공

### 4.2 PlayMCP MCP 서버 (메모 활용)
- AI 대화에서 메모 검색
- 카테고리별 조회
- 요약 리포트 생성

### 4.3 나에게 보내기 (결과 전달)
- 정리된 메모를 카톡으로 전송
- 사용자의 "나와의 채팅방"에 저장됨

---

## 5. 기술 스택

| 컴포넌트 | 기술 |
|----------|------|
| 스킬서버 | FastAPI (Python) |
| MCP 서버 | FastAPI (Python) |
| 호스팅 | Vercel |
| DB | SQLite → Supabase |
| AI | Kanana API |
| 카카오 | 채널 챗봇, 나에게 보내기 API |

---

## 6. 공모전 차별화 포인트

### 심사 기준 대응

1. **서비스 안정성** ✅
   - Vercel 서버리스로 안정적 운영
   - 에러 핸들링 철저

2. **편의성** ✅
   - 저장만 하면 자동 정리
   - 습관 변화 최소화
   - 자연어 검색

3. **창의성** ✅
   - "나와의 채팅방 막 던지기" 습관을 AI로 해결
   - 카카오 생태계 시너지 (채널 + 챗봇 + 톡 + AI)
   - 기존에 없는 서비스
